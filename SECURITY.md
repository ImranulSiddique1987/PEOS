# Security Policy

Thank you for helping keep PEOS secure.

The security of PEOS is a top priority. We appreciate responsible disclosure of security vulnerabilities and will work to investigate and resolve legitimate reports as quickly as possible.

---

# Supported Versions

The following table describes the current support policy.

| Version                | Supported |
| ---------------------- | --------- |
| Unreleased Development | ✅ Yes    |
| Future Stable Releases | ✅ Yes    |
| End-of-Life Releases   | ❌ No     |

---

# Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub Issues.

Instead, report them privately using the designated security contact that will be published before the first public release.

Until an official security contact is available, please keep vulnerability details confidential.

---

# What to Include

Please include as much information as possible:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Affected components
- Proof of Concept (if applicable)
- Suggested mitigation (optional)

Providing complete information helps us investigate and resolve issues more efficiently.

---

# Response Process

Once a report is received, the PEOS team will:

1. Acknowledge receipt of the report.
2. Assess the reported issue.
3. Determine severity and impact.
4. Develop and validate a fix.
5. Coordinate responsible disclosure when appropriate.
6. Publish the fix in a future release.

---

# Security Practices

PEOS uses multiple layers of security throughout the development lifecycle, including:

- GitHub CodeQL static analysis
- Dependabot dependency monitoring
- Continuous Integration (CI)
- Conventional Commit enforcement
- Changesets-based release management
- Code review before merge

Additional security controls will be introduced as PEOS evolves.

---

# Responsible Disclosure

Please provide the PEOS team with a reasonable opportunity to investigate and remediate any reported vulnerability before publicly disclosing details.

We greatly appreciate responsible disclosure and the efforts of the security community.

---

# Future Enhancements

Future versions of PEOS will introduce:

- Dedicated security contact
- Security advisories
- CVE management (if applicable)
- Vulnerability disclosure program
- Security release process

---
